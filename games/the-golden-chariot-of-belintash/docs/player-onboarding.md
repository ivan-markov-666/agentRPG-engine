# Player Onboarding · The Golden Chariot of Belintash

## 1. Flow Overview
Before започване на каквато и да е сцена, Game Master-ът трябва да проведе кратко интервю с играча и да фиксира резултатите в `player-data/session-init.json`. Задължителните полета са:

| Поле | Описание | Източник |
| --- | --- | --- |
| `preferred_language` | Езикът, на който LLM ще води наратива. | Избор на играча (всеки език, поддържан от LLM). |
| `player_gender` | Как героят да бъде адресиран (виж §3). | Избор на играча. |
| `player_gender_custom` | Ръчен текст, ако `player_gender = custom`. | По желание на играча. |
| `player_name` | Личният/геройският псевдоним. | Избор на играча (виж §4 за препоръки). |

> **Важно:** Сесията не стартира, докато горните полета не са попълнени и записани. GM-ът трябва да пази тези стойности за бъдещи сесии или NG+ run-ове.

## 2. Стъпка 1 — Избор на език
1. Попитай играча на кой език иска да води кампанията. Приемай всякакъв език, поддържан от LLM (пример: `bg`, `en`, `de`, `ar`, `es`).
2. Обясни, че въпреки че документите са на български/английски, целият диалог и описания ще бъдат адаптирани към избрания език.
3. Запиши стойността в `preferred_language` и (по избор) стартирай помощния скрипт:
   ```bash
   npm run lang:set -- --game the-golden-chariot-of-belintash --language <ISO код>
   ```
4. Ако езикът изисква допълнителен контекст (напр. диалект или формален тон), добави бележка в `player-data/runtime/state.json > content_sets.base-campaign.notes`.

## 3. Стъпка 2 — Избор на пол
| Опция (`player_gender`) | Как да я представиш |
| --- | --- |
| `male` / `female` | Стандартни обръщения, подходящи за XIII век ("комит", "дева", "воин" и т.н.). |
| `nonbinary` | Нейтрални обръщения („пазителя“, „избраника“) + избягване на родово маркирани прилагателни. |
| `custom` | Играчът диктува предпочитания текст (например "духът", "калакан" и т.н.). Задължително попълни `player_gender_custom`. |

> При промяна на предпочитанията между сесиите: обнови `session-init.json` и остави бележка в `runtime/state.json -> player_profile` (виж §5).

## 4. Стъпка 3 — Избор на име
Предложи 3–5 тематични имена според избрания пол, но подчертавай, че това са само идеи.

**Примерни имена (XIII в., Родопи):**

| Male | Female | Neutral/Unisex |
| --- | --- | --- |
| Симеон | Десислава | Зорница |
| Калоян | Калина | Мирослав(а) |
| Добромир | Теодора | Радостин(а) |
| Иванко | Елена | Вяра |
| Асен | Ефросина | Лада |

Играчът може да въведе напълно различно име (вкл. чуждоезиково или кодово). Запиши точната форма в `player_name`. Ако името съдържа специфична транслитерация, копирай я дословно.

## 5. Записване на профила
1. Обнови `player-data/session-init.json` със следното:
   ```json
   {
     "player_name": "<име>",
     "player_gender": "<male|female|nonbinary|custom>",
     "player_gender_custom": "<само ако custom>",
     "preferred_language": "<код>",
     "debug": false
   }
   ```
2. По избор синхронизирай резюме в `player-data/runtime/state.json`:
   ```json
   "player_profile": {
     "name": "<име>",
     "gender": "<опция>",
     "language": "<код>",
     "notes": "Нужди от обръщение/тон"
   }
   ```
3. Ако играчът иска името/пола да влияят на NPC реакциите, използвай `content_sets.base-campaign.state.notes` или специфични quest flags.

## 6. Стартиране на играта
- След като профилът е записан, GM-ът може да зареди сцената (CLI, UI или tabletop driver).
- Всяко следващо продължение (save/load) трябва да проверява `session-init.json`, за да гарантира консистентност на езика, пола и името.
- При NG+/DLC кампания използвай същия файл като източник или събирай нов профил, ако играчът го изисква.
